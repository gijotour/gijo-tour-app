# 지아이조 투어 (GIJO TOUR) B2B 포털 스토리보드

## 1. 개요 (Overview)
- **서비스 명:** GIJO TOUR
- **서비스 성격:** 여행 설계사와 비즈니스 파트너를 연결하는 하이엔드 B2B 투어 플랫폼
- **디자인 컨셉:** 엘리트(Elite), 프리미엄, 다크 모드, 글래스모피즘(Glassmorphism)

---

## 2. 화면별 상세 구성 (Scene Breakdown)

### Scene #1: 메인 랜딩 페이지 (Hero Section)
- **목적:** 브랜드 정체성 확립 및 시각적 압도
- **주요 요소:**
  - 시네마틱 배경 영상/이미지
  - "Elite Business Travel" 브랜드 슬로건
  - 주요 서비스(B2B 특전) 소개 섹션
- **전환:** 스크롤 시 '설계사 쇼케이스'로 연결

### Scene #2: 설계사 쇼케이스 & 지역 선택 (Showcase & Selector)
- **목적:** 실제 여행 상품(제안서) 탐색
- **주요 요소:**
  - 지역별 필터 (필리핀, 베트남, 태국, 라오스 등)
  - 제안서 카드: 설계사 프로필, 평점, 주요 일정, 가격 정보
  - 마우스 오버 시 상세 정보 및 리뷰 팝업
- **액션:** '상세 보기' 클릭 시 결제 또는 문의로 연결

### Scene #3: 설계사 TV (Designer TV)
- **목적:** 영상 기반의 신뢰도 확보 및 현장감 전달
- **주요 요소:**
  - 그리드 형태의 비디오 갤러리
  - 현지 투어 현장 및 호텔 리뷰 영상
- **기능:** 비디오 모달 재생 기능

### Scene #4: 로그인 페이지 (Authentication)
- **목적:** 권한별(관리자/설계사) 대시보드 진입
- **주요 요소:**
  - 역할 선택 버튼 (Admin / Designer)
  - 미니멀한 로그인 폼

### Scene #5: 관리자 패널 (Admin Panel)
- **목적:** 플랫폼 전체 운영 및 통계 관리
- **주요 요소:**
  - 실시간 통계 대시보드 (누적 평점, 고객 만족도 등)
  - 등록된 설계사 리스트 관리 (편집/삭제)
- **특징:** 모든 데이터 변동 사항은 Mock DB 및 LocalStorage 반영

### Scene #6: 설계사 대시보드 (Designer Dashboard)
- **목적:** 설계사가 직접 상품(제안서)을 등록하고 관리
- **주요 요소:**
  - 새로운 제안서 등록 폼 (지역, 일정, 가격, 상세 설명)
  - 본인이 등록한 제안서 리스트 및 상태 확인

### Scene #7: 결제 및 예약 (Payment Page)
- **목적:** 투어 예약 및 결제 프로세스 완료
- **주요 요소:**
  - 선택한 패키지 정보 요약
  - 예약 정보 입력 폼
  - 결제 수단 선택 (PayPal MCP 연동 기반 레이아웃)

---

## 3. 사용자 흐름 (User Flow)
1. **일반 사용자:** 홈 접속 -> 지역 필터링 -> 상품 탐색 -> 상세 확인 -> 결제 
2. **설계사:** 로그인 -> 디바이스보드 접속 -> 신규 제안서 작성 -> 상품 노출 확인
3. **관리자:** 로그인 -> 관리자 패널 접속 -> 운영 지표 확인 -> 설계사 활동 관리

---

## 4. 기술적 포인트
- **UI Scaling:** 사용자 선호에 따른 전체 UI 크기 조절 기능 (SizeControl)
- **Responsive:** 모바일 및 태블릿 최적화 레이아웃
- **Persistence:** LocalStorage를 활용한 상태 유지 (로그인 상태, 모델 로드 상태 등)
